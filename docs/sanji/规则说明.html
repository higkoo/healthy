Wiki 规则说明（SCHEMA）

类型: schema（Wiki 规则配置） 创建时间: 2026-07-21 最后更新: 2026-07-21 适用范围: reji-wiki 整个知识库

一、建库目的

本 Wiki 以 原始资料/ 目录下的全部 Markdown 文件（由倪海厦先生完整教学与临床资料 PDF/DOC 全文提取）为原始资料，按 LLM Wiki（Andrej Karpathy 模式）“编译"为结构化、可增量更新的中文 Markdown 知识库。目标是把零散的 PDF / DOC / DOCX 资料沉淀为可检索、可关联、可长期保鲜的个人知识库。

二、目录结构

reji-wiki/
├── index.md          ← 内容索引（保留英文文件名，便于工具识别）
├── 变更日志.md        ← 变更时间线
├── 规则说明.md        ← 本文件（对应原 SCHEMA.md）
├── 原始资料/          ← 第一层：原始资料清单（不存放副本，仅建引用清单）
│   └── 资料清单.md
├── 实体/             ← 第二层：人物、机构、作品等实体页
├── 概念/             ← 第二层：理论、技法、方法论等概念页
└── 主题/             ← 第二层：综述、对比、汇编等主题页

说明：原始 PDF/DOC 体积大（总计约 1.5GB），已全部转换为 Markdown 格式存放于 原始资料/ 目录（81 个文件）。Wiki 页面通过相对路径链接引用这些 raw Markdown 文件，如 链接文字 指向 ../原始资料/文件名.md（子目录页面）或 原始资料/文件名.md（根目录页面）。原始 PDF/DOC 位于 ../../docs/NiHaixia/（只读，不修改）。

三、命名规范

  • 保留英文的文件/目录：仅 index.md 一个文件名保留英文（按用户要求）；其余文件与目录一律使用中文命名（如 实体/倪海厦.md、概念/伤寒论.md）。
  • 页面命名：使用简短中文主题词，必要时以"概念/经方.md"形式避免重名。
  • 链接格式：一律使用纯 Markdown 相对路径链接（不使用 <> 包裹、不使用 [[wikilink]]）。例如 [倪海厦](实体/倪海厦.md)、[人纪](概念/人纪.md)。
    • 内部 Wiki 链接：路径含中文可直接书写（可读性好），如 [针灸](概念/针灸.md)。
    • 回链原始资料：根目录页面用 原始资料/<文件名>，子目录页面用 原始资料/<文件名>；因原始文件名含中文与括号，按 URL 规范做百分号编码（Markdown 渲染时会正确解析），如 [专题-乳癌篇-倪海厦.md](原始资料/专题-乳癌篇-倪海厦.md)。

四、页面模板

每个 Wiki 页面遵循以下结构：

# {页面标题}

> **类型**: 实体 | 概念 | 主题 | 对比
> **创建时间**: YYYY-MM-DD
> **最后更新**: YYYY-MM-DD
> **来源**: [资料清单](原始资料/资料清单.md) 或具体原文链接

## 摘要
{一段话概括核心内容}

## 详情
{详细内容，可含多个子标题}

## 关联
- 相关实体: [倪海厦](实体/倪海厦.md)
- 相关概念: [经方](概念/经方.md)
- 参见: [人纪系列总览](主题/人纪系列总览.md)

## 引用来源

## 变更记录
- YYYY-MM-DD: 初始创建

五、知识质量标准

  • 不臆造：所有内容须来源于 原始资料/ 中的 raw Markdown 文件，或标注为"AI 综合推理”。
  • 必标注来源：每个知识点都给出原文链接。
  • 标注矛盾：不同资料对同一问题说法不一致时，用 > ⚠️ 矛盾标注 引用块显式标记。
  • 增量更新：更新已有页面时保留历史、追加新内容，并在变更记录中注明。

六、操作纪律

  • 原始资料目录只读，绝不修改 ../../docs/NiHaixia/ 下的任何原始 PDF/DOC 文件。
  • 每次 ingest 后必须更新 index.md 与 变更日志 。
  • 所有页面均登记进索引，避免孤立页。

七、重要提示

本库内容均出自倪海厦先生公开教学资料与民间校勘本，仅供中医学习与研究参考，不构成医疗建议。如有身体不适，请务必前往正规医疗机构就诊。原始资料中多处声明"仅供网络测试、请购买原版"，使用本 Wiki 亦请遵守该原则。
